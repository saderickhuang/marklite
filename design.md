# WPF Markdown Editor - 项目设计方案

## 1. 项目概述

**项目名称**：MarkLite (暂定)
**技术栈**：WPF (.NET 8) + C#
**定位**：轻量级 Markdown 编辑器，参考 Typora 的"即时渲染"模式

---

## 2. 核心特性（参考 Typora）

### 2.1 编辑模式
| 特性 | 描述 |
|-----|------|
| **即时渲染** | 所见即所得，边写边渲染，无需切换模式 |
| **源码模式** | 纯文本编辑，实时预览 |
| **专注模式** | 隐藏所有 UI，仅保留编辑区域 |

### 2.2 Markdown 支持
- CommonMark + GFM（GitHub Flavored Markdown）
- 代码高亮（语法高亮）
- 数学公式（LaTeX / KaTeX）
- 表格、任务列表、目录自动生成
- 脚注、上标、下标

### 2.3 文件操作
- 新建/打开/保存/另存为
- 自动保存
- 最近文件列表
- 拖拽打开

### 2.4 导出
- PDF
- HTML
- Word (DOCX)
- 图片

---

## 3. 技术架构

### 3.1 分层架构

```
┌─────────────────────────────────────────┐
│              UI 层 (WPF)                │
│  - MainWindow / EditorControl           │
│  - 主题系统 (CSS/Style)                 │
├─────────────────────────────────────────┤
│           ViewModel 层 (MVVM)           │
│  - MainViewModel                        │
│  - EditorViewModel                      │
│  - SettingsViewModel                    │
├─────────────────────────────────────────┤
│            业务逻辑层                    │
│  - MarkdownParser (解析器)              │
│  - FileService (文件操作)               │
│  - ExportService (导出服务)             │
│  - ThemeService (主题管理)              │
├─────────────────────────────────────────┤
│              核心库                      │
│  - Markdig (Markdown解析)               │
│  - AvalonEdit (代码编辑)                │
│  - WebView2 (渲染HTML)                 │
└─────────────────────────────────────────┘
```

### 3.2 关键技术选型

| 组件 | 选型 | 理由 |
|-----|------|------|
| **Markdown 解析** | Markdig | .NET 最强 Markdown 库，支持扩展 |
| **代码编辑** | AvalonEdit | WPF 原生，性能好 |
| **HTML 渲染** | WebView2 | Chromium 内核，兼容性好 |
| **数学公式** | KaTeX | 比 MathJax 快 |
| **主题系统** | CSS + WPF Style | 与 Typora 一致 |

### 3.3 核心模块设计

#### EditorControl (编辑器控件)
```
职责：处理用户输入、即时渲染切换
- 输入事件捕获
- Markdown 语法检测
- 渲染触发（防抖 300ms）
- 光标位置同步
```

#### MarkdownRenderer (渲染器)
```
职责：将 Markdown 转为 HTML 并展示
- 调用 Markdig 解析
- 注入 KaTeX / 代码高亮
- 注入 CSS 主题
- 输出到 WebView2
```

#### FileService (文件服务)
```
职责：文件读写、自动保存
- FileSystemWatcher 监控
- 增量保存
- 编码检测 (UTF-8 BOM)
```

## 7. 风险与挑战

1. **WebView2 性能** - 大文档渲染可能卡顿，需优化
2. **光标同步** - 编辑区与预览区光标位置对齐
3. **主题兼容性** - CSS 在 WebView2 中的兼容性问题
4. **导出 PDF** - 需要额外处理分页
